Great choice! Adding SSH access to your Cisco switch and router increases security over insecure protocols like Telnet. Let’s walk through configuring SSH v2 with local login on both the switch and router.

✅ SSH Configuration Overview

We'll do the following on both the switch and router:

  1. Set a hostname and domain name (required for RSA key generation).

  2. Create a local user with a password.

  3. Generate RSA keys.

  4. Enable SSH version 2.

  5. Configure the vty lines to use SSH only and login locally.

⚙️ On Switch1 – SSH Server Setup

bash
enable
configure terminal
hostname Switch1
ip domain-name mynet.local

! Create local user
username admin privilege 15 secret P@ssw0rd

! Generate RSA keys
crypto key generate rsa
! When prompted, choose key size – type: 1024

! Enable SSH v2 only
ip ssh version 2

! Enable local login and SSH-only on VTY lines
line vty 0 4
 login local
 transport input ssh
exit

! (Optional) Set idle timeout and banner
exec-timeout 5 0
banner motd ^Authorized Access Only!^

exit
write memory

configure terminal

! Create local user

ip ssh version 2

! Secure VTY access

Great! Adding Port Security on your access switch interfaces will protect your network from:

  • MAC flooding attacks

  • Unauthorized device connections

  • Rogue DHCP servers

Let’s apply Port Security to access ports on SW1 and SW2 where end-user devices (PCs) connect.

✅ Port Security Goals:

  • Allow only one MAC address per port.

  • Automatically learn and stick the MAC address.

  • If a violation occurs, set the port to shutdown (or restrict/protect, if preferred).

  • Keep trunk ports (e.g., EtherChannel, uplinks) unaffected.

🔧 Apply Port Security on Access Ports

Assume:

  • On SW1, PC0 is connected to Fa0/1, and PC1 to Fa0/2.

  • On SW2, PC2 is on Fa0/1, PC3 on Fa0/2.

⚙️ Configuration on SW1:

bash
interface range fastEthernet 0/1 - 2
 switchport mode access
 switchport port-security
 switchport port-security maximum 1
 switchport port-security violation shutdown
 switchport port-security mac-address sticky
exit

⚙️ Configuration on SW2:

bash
interface range fastEthernet 0/1 - 2
 switchport mode access
 switchport port-security
 switchport port-security maximum 1
 switchport port-security violation shutdown
 switchport port-security mac-address sticky
exit

🔍 Verify Port Security

You can check the status and learned MACs:

bash
show port-security interface fastEthernet 0/1
show port-security address

🚨 Simulate a Port Security Violation

  1. Disconnect PC0 from SW1 Fa0/1.

  2. Connect a different PC (with a different MAC) to that same port.

  3. Port will shut down due to violation.

To recover:

bash
interface fastEthernet 0/1
 shutdown
 no shutdown

💡 Optional Modes for Violation

ModeBehavior
shutdown (default)Port goes into error-disabled (shuts down)
restrictDrops packets from unknown MACs, logs violation
protectSilently drops packets from unknown MACs

You can change the mode like:

bash
switchport port-security violation restrict

✅ Summary

You’ve now secured your network edge:

  • Only authorized devices are allowed on access ports.

  • Any new device will trigger a violation.

  • The switch automatically learns and remembers MACs with sticky.
