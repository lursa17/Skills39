Great choice! Adding SSH access to your Cisco switch and router increases security over insecure protocols like Telnet. Let’s walk through configuring SSH v2 with local login on both the switch and router.

✅ SSH Configuration Overview

We'll do the following on both the switch and router:

  1. Set a hostname and domain name (required for RSA key generation).

  2. Create a local user with a password.

  3. Generate RSA keys.

  4. Enable SSH version 2.

  5. Configure the vty lines to use SSH only and login locally.

⚙️ On Switch1 – SSH Server Setup

bash
enable
configure terminal
hostname Switch1
ip domain-name mynet.local

! Create local user
username admin privilege 15 secret P@ssw0rd

! Generate RSA keys
crypto key generate rsa
! When prompted, choose key size – type: 1024

! Enable SSH v2 only
ip ssh version 2

! Enable local login and SSH-only on VTY lines
line vty 0 4
 login local
 transport input ssh
exit

! (Optional) Set idle timeout and banner
exec-timeout 5 0
banner motd ^Authorized Access Only!^

exit
write memory

⚙️ On Router1 – SSH Server Setup

bash
enable
configure terminal
hostname Router1
ip domain-name mynet.local

! Create local user
username admin privilege 15 secret P@ssw0rd

! Generate RSA keys
crypto key generate rsa
! Choose 1024-bit or 2048-bit key

! Enable SSH version 2
ip ssh version 2

! Secure VTY access
line vty 0 4
 login local
 transport input ssh
exit

! (Optional) MOTD and timeout
exec-timeout 5 0
banner motd ^Authorized Router Access Only!^

exit
write memory

🧪 Testing SSH Access in Packet Tracer

  1. From any PC in the network (e.g., PC0):

    • Go to Desktop > Terminal

    • Type:

      bash
      ssh -l admin 192.168.10.1

    • Or to access the switch:

      bash
      ssh -l admin 192.168.99.2

  2. Enter the password: P@ssw0rd

You should now see the Router1> or Switch1> prompt.
