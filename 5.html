VTP & Etherchannel

ChatGPT said:

Excellent! Let’s extend your existing topology with:

🎯 New Goals:

  1. Add multiple switches and implement VTP (VLAN Trunking Protocol) to manage VLANs centrally.

  2. Use EtherChannel to bundle trunk links between switches for:

    • Increased bandwidth.

    • Redundancy.

  3. Maintain SSH access and VLAN segmentation from earlier.

🖥️ Extended Topology Overview:

pgsql
         PC0(VLAN10)     PC1(VLAN20)
              |              |
           Access Ports    Access Ports
                |              |
             [SW1] ←────────────→ [SW2]
                ||           ||
                ||<-- EtherChannel -->||
                ||           ||
             [Router1] ←→ [Router2 / Internet]

  • SW1 = VTP Server

  • SW2 = VTP Client

  • SW1 ↔ SW2 = EtherChannel (Port-channel)

  • Router1 still handles Router-on-a-Stick, DHCP, NAT, EIGRP, SSH, etc.

🧾 Configuration Steps

✅ 1. Configure VTP on Switch1 (Server)

bash
enable
configure terminal
hostname SW1

vtp domain MYSCHOOL
vtp mode server
vtp password cisco123

! Create VLANs (will be advertised to SW2)
vlan 10
 name Students
exit
vlan 20
 name Staff
exit

✅ 2. Configure VTP on Switch2 (Client)

bash
enable
configure terminal
hostname SW2

vtp domain MYSCHOOL
vtp mode client
vtp password cisco123

📝 You do not create VLANs on SW2 — they'll sync from SW1.

✅ 3. Set Up EtherChannel Between SW1 and SW2

On SW1:

bash
interface range fastEthernet 0/3 - 4
 switchport mode trunk
 channel-group 1 mode active
exit

interface port-channel 1
 switchport mode trunk

On SW2:

bash
interface range fastEthernet 0/3 - 4
 switchport mode trunk
 channel-group 1 mode passive
exit

interface port-channel 1
 switchport mode trunk

📝 We're using LACP here:

  • active (SW1) and passive (SW2) form a dynamic EtherChannel.

✅ You should see the bundle form automatically:

bash
show etherchannel summary

✅ 4. VLAN Access Port Assignments

On SW1:

bash
interface fastEthernet 0/1
 switchport mode access
 switchport access vlan 10

interface fastEthernet 0/2
 switchport mode access
 switchport access vlan 20

On SW2:

Same as above (VLANs were synced via VTP):

bash
interface fastEthernet 0/1
 switchport mode access
 switchport access vlan 10

interface fastEthernet 0/2
 switchport mode access
 switchport access vlan 20

✅ 5. Trunk to Router (Router-on-a-Stick)

Ensure SW1 → Router1 is a trunk (already done in previous steps):

bash
interface gigabitEthernet 0/1
 switchport mode trunk

🧪 6. Verify and Test

From SW2:

bash
show vtp status        ← VTP mode should be CLIENT
show vlan brief        ← VLAN 10/20 should appear
show etherchannel summary

From PCs, try:

  • ping 192.168.10.1 (from VLAN10)

  • ping 192.168.20.1 (from VLAN20)

  • ssh admin@192.168.99.2 (switch)

  • ssh admin@192.168.10.1 (router)
