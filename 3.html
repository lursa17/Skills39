Awesome! Let’s continue by simulating internet access and extending the network with a second router, using EIGRP for dynamic routing and NAT for internet access. This will complete a full LAN-to-WAN enterprise simulation.

less

                |                    |
         [Access Ports on SWITCH]

                  [Router1]
               G0/0.10   G0/0.20
                 |          |
              VLAN10    VLAN20
                 |
               G0/1 (LAN) --- G0/0 (WAN) --- Router2 (Edge/ISP)
                                  |
                              203.0.113.1
                                  |
                               [Cloud]

✅ GOALS:

  1. Add Router2 as WAN/Internet Router

  2. Use EIGRP between Router1 and Router2

  3. Use NAT on Router1 to translate internal IPs

  4. Simulate Internet with a Cloud and a DNS/HTTP server

🧾 CONFIGURATION STEPS

⚙️ Step 1: Connect Devices

  • Router1 G0/1 → Router2 G0/0

  • Router2 G0/1 → Cloud (add a generic server here as public DNS/web)

⚙️ Step 2: Configure Router2 (ISP/Edge Router)

bash
enable
configure terminal
hostname Router2

interface gigabitEthernet 0/0
 ip address 10.0.0.2 255.255.255.0
 no shutdown
exit

interface gigabitEthernet 0/1
 ip address 203.0.113.1 255.255.255.0
 no shutdown
exit

router eigrp 100
 network 10.0.0.0
 no auto-summary
exit

ip route 192.168.0.0 255.255.0.0 10.0.0.1  ← static back-route
write memory

⚙️ Step 3: Update Router1 (add WAN + NAT)

bash
enable
configure terminal

! Configure interface to ISP
interface gigabitEthernet 0/1
 ip address 10.0.0.1 255.255.255.0
 no shutdown
exit

! Update EIGRP to include WAN link
router eigrp 100
 network 10.0.0.0
exit

! Define NAT inside (LAN interfaces)
interface gigabitEthernet 0/0.10
 ip nat inside
exit
interface gigabitEthernet 0/0.20
 ip nat inside
exit

! Define NAT outside (WAN)
interface gigabitEthernet 0/1
 ip nat outside
exit

! Create NAT overload rule
access-list 1 permit 192.168.0.0 0.0.255.255
ip nat inside source list 1 interface gigabitEthernet 0/1 overload

! Default route to ISP
ip route 0.0.0.0 0.0.0.0 10.0.0.2

write memory

⚙️ Step 4: Configure the Cloud/DNS/HTTP Server

  1. Add a Server device (name: PublicServer).

  2. Assign:

    • IP: 203.0.113.100

    • Subnet: 255.255.255.0

    • Gateway: 203.0.113.1

  3. On Services tab:

    • Enable HTTP and DNS

    • Set domain: example.com pointing to 203.0.113.100

🧪 Step 5: Testing Internet Access

From PC0 or PC1:

bash
ping 8.8.8.8                ← Simulate Google DNS (optional)
ping 203.0.113.1            ← Ping Router2
ping 203.0.113.100          ← Ping Public Server
nslookup example.com        ← Should resolve

Open Web Browser on PC0:

  • Go to: http://example.com

  • You should see the server’s page!

🧩 Final Notes

  • You now have DHCP + VLANs + Inter-VLAN Routing + EIGRP + NAT + Internet Simulation.

  • This simulates a real-world enterprise network with WAN connectivity.
