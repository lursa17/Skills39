Perfect! Since you're using Cisco Packet Tracer and configuring a switch and router, let’s walk through a basic network setup example:

🔧 Scenario:

  • A router is connected to a switch.

  • PCs are connected to the switch.

  • You want to:

    • Assign IP addresses.

    • Configure VLAN 1 on the switch.

    • Set up the router with an IP.

    • Make sure PCs can ping the router (basic connectivity).

🖥️ Example Topology:

lua
PC0 ----
         \
          SWITCH ---- ROUTER ---- (could be internet or another network)
         /
PC1 ----

✅ Step 1: Assign IP Addresses (Plan)

DeviceInterfaceIP AddressSubnet Mask
RouterG0/0192.168.1.1255.255.255.0
Switch VLAN1VLAN1192.168.1.2255.255.255.0
PC0NIC192.168.1.10255.255.255.0
PC1NIC192.168.1.11255.255.255.0
Default GWPCs' gateway192.168.1.1

⚙️ Step 2: Configure the Switch

Click on the switch in Packet Tracer > Go to CLI tab and enter:

bash
enable
configure terminal
hostname Switch1
interface vlan 1
 ip address 192.168.1.2 255.255.255.0
 no shutdown
exit
ip default-gateway 192.168.1.1
exit
write memory

⚙️ Step 3: Configure the Router

Click the router > CLI:

bash
enable
configure terminal
hostname Router1
interface gigabitEthernet 0/0
 ip address 192.168.1.1 255.255.255.0
 no shutdown
exit
exit
write memory

⚙️ Step 4: Set IPs on PCs

Click PC0 > Desktop tab > IP Configuration:

  • IP: 192.168.1.10

  • Subnet Mask: 255.255.255.0

  • Default Gateway: 192.168.1.1

Do the same for PC1, use IP: 192.168.1.11.

🧪 Step 5: Test Connectivity

  • From PC0, open the command prompt and type:

    bash
    ping 192.168.1.1  ← (router)
ping 192.168.1.2  ← (switch)
ping 192.168.1.11 ← (PC1)
